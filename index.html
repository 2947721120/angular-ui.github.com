jQuery Passthrough

What?

Fires a jQuery command specified in the expression on an element.

Why?

Not every jQuery plugin requires creating a new directive just to use it. Instead, use the pass-through directive to rapidly port functionality. It is probably the best solution for 75% of the cases you will encounter.

If a plugin requires more robust integration with AngularJs then you may need to look into creating a custom directive instead, or contact us with a request.

How?

<input value="Click Here to show the jQuery UI Datepicker" ui-jq="datepicker">

You can also add a ui-options attribute that will be evaluated and passed to the command.

You will also be able to create an app-wide configuration so that you aren't forced to manually pass options every time you call the directive.

Scrollfix

What?

Add a 'ui-fixed' class to elements when the page scrolls past them

They see me rollin...

Why?

Make elements sticky, or simply appear different after scrolling past a certain point

How?

<p ui-scrollfix>They see me rollin...</p>

You can optionally pass a number to ui-scrollfix which would override the detected y-offset of the element

Select2

Select2 is inspired by Chosen

What?

Value is: {{select2}} (choose second)

Why?

This would work fine using the jQuery passthrough alone, but this directive adds support for changing the ng-model directly. If the value is programmatically updated, then select2 is automatically updated to reflect the changes.

How?

<p>Value is: {{select2}} <a ng-click="select2='two'">(choose second)</a></p>
<select ui-select2 ng-model="select2">
	<option value="">Pick a number</option>
	<option value="one">First</option>
	<option value="two">Second</option>
	<option value="three">Third</option>
</select>

Show, Hide and Toggle Alternatives

What?

Instead of doing ng-show and ng-hide which simply sets display:block/none a ui-show and ui-hide class will be added which can give you tighter control over how things appear or disappear.

Toggle State: {{!!showHide}} Show Hide Toggle

Why?

CSS3 transitions of course! Applying a class means you can specify exactly how these two states behave. In addition, the show/hide variants do not enforce css rules when they are false (unless you use toggle), so the default CSS can still apply.

Toggle State: {{!!showHideAn}}

Show Hide Toggle

But can't you just do ng-class="someExpression&&'ui-show'"?

... shutup.

How?

<p><a ng-click="showHide=!showHide">Toggle State: {{!!showHide}}</a></p>
<div ui-show="showHide">Show</div>
<div ui-hide="showHide">Hide</div>
<div ui-toggle="showHide">Toggle</div>

Length

What?

Get the length attribute of an array or string

There are {{items | filter:lengthFilter | length}} items.

  • {{value}}

Why?

If you used another filter to modify the array or string, you cannot look at its inherit properties. This allows you to get the length property.

How?

<p><input ng-model="lengthFilter" placeholder="Filter the list"></p>
<p>There are {{items | filter:lengthFilter | length}} items.</p>
<ul>
	<li ng-repeat="value in items | filter:lengthFilter">{{value}}</li>
</ul>
...
$scope.items = ['First','Second','Third','Fourth','Fifth'];
